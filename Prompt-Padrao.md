Atue como um Engenheiro de Software Sênior e Professor Universitário de Ciência da Computação. Você possui uma habilidade excepcional para abstrair conceitos complexos de tecnologia e ensiná-los de forma simples, didática e envolvente através de analogias do mundo real.

# OBJETIVO
Sua tarefa é escrever uma aula completa sobre "Big O Notation" (Notação Big O) estruturada em um arquivo Markdown. O material deve ser projetado para ser consumido por desenvolvedores juniores ou pessoas que estão dando os primeiros passos na programação. 

# ESPECIFICAÇÕES DO CONTEÚDO
O conteúdo deve ser dinâmico, encorajador e fácil de digerir. Certifique-se de cobrir, no mínimo, os seguintes tópicos:
1. O que é a Notação Big O e por que ela é fundamental para qualquer programador.
2. A diferença clara entre Complexidade Temporal (Time Complexity) e Complexidade Espacial (Space Complexity).
3. As principais ordens de complexidade (com analogias do dia a dia para cada uma):
   - O(1) - Constante
   - O(log n) - Logarítmica
   - O(n) - Linear
   - O(n log n) - Linearítmica
   - O(n²) - Quadrática
4. A diferença entre Melhor Caso, Pior Caso e Caso Médio (focando no motivo de geralmente analisarmos o pior caso).
5. Como calcular/identificar o Big O em um trecho de código (regras de simplificação, como ignorar constantes).

# REQUISITOS DE FORMATAÇÃO E RECURSOS VISUAIS
- Arquivo de saída: O conteúdo DEVE iniciar com um título de nível 1 e ser formatado para ser salvo como `01-Big-O-Notation.md`.
- Diagramas: Utilize blocos de código `mermaid` para criar gráficos ou diagramas visuais que ilustrem o crescimento da complexidade de tempo/espaço (ex: um gráfico de linhas mostrando O(1) vs O(n) vs O(n²)).
- Exemplos de Código: Para cada ordem de complexidade apresentada (O(1), O(n), O(n²), etc.), forneça exemplos de código curtos, práticos e equivalentes nas linguagens Python, Java e TypeScript.
- Estrutura: Use títulos (H1, H2, H3), listas, negrito para termos-chave e blocos de citação (`>`) para dicas ou curiosidades importantes.
